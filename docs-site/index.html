Why the-one-mcp?

Unlimited memory

Your AI assistant no longer forgets what it learned in the last turn. Every doc, every screenshot, every code chunk stays searchable across sessions.

Hybrid search

Dense embeddings catch semantic intent; BM25 / SPLADE sparse vectors catch exact identifiers. Both run in parallel, fused with configurable weights.

13-language code chunker

Tree-sitter-backed AST chunking for Rust, Python, TS/JS, Go, Swift, Ruby, C, C++, Java, Kotlin, PHP, and Zig. Symbol names, signatures, and line ranges ride along with every chunk.

Multimodal

Embed and search screenshots and diagrams alongside text. Optional OCR extracts text so you can find that whiteboard photo months later.

Tool catalog

365 curated CLI tools across 10 languages — pytest, eslint, clippy, ruff, clang-tidy, and many more. Ask "what test runner should I use here?" and get a grounded answer.

Auto-reindex

A built-in file watcher keeps your index fresh as you edit. Change a markdown file or drop in a new screenshot — it is searchable within seconds.

Install in one command

curl -fsSL https://raw.githubusercontent.com/michelabboud/the-one-mcp/main/scripts/install.sh | bash

The installer downloads the right binary for your platform, sets up the default config, and registers the-one-mcp with Claude Code, Gemini CLI, OpenCode, and Codex automatically.

Supported: Linux x86_64 · Linux aarch64 · macOS Intel (lean) · macOS Apple Silicon · Windows x86_64 · Windows aarch64

Benchmarks

We ship a retrieval benchmark suite that compares four configurations (dense, dense + rerank, hybrid, full pipeline) against three query sets (exact, semantic, mixed) on the-one-mcp's own source tree. See the raw numbers in benchmarks/results.md.

The benchmark is not in CI (it requires a running Qdrant), so we publish numbers from reproducible local runs. Run it yourself with cargo run --release --example retrieval_bench -p the-one-memory --features tree-sitter-chunker.
